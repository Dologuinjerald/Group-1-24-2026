# Laboratory 1: Tools & Architecture Setup

## Description

This laboratory activity focuses on setting up the development environment and organizing the project structure properly. A local Git repository is initialized to manage version control, while the folder architecture is designed to separate components, pages, assets, and documentation.

## Folder Structure

* **src/** – main source files
* **components/** – reusable components
* **pages/** – application pages
* **assets/** – images and static files
* **docs/** – project documents

## Component Layering Strategy

The project uses a layered structure where reusable components are separated from main pages to improve maintainability, readability, and future scalability.

## Tools Used

* **Termux** for terminal commands
* **GitHub** for version control
* **Visual Studio Code** for editing source code
